Git

Git is software for tracking changes in any set of files, usually used for coordinating work among programmers collaboratively developing source code during software development. Its goals include speed, data integrity, and support for distributed, non-linear workflows

Install  Git

$ git config --global user.name "John Doe"
$ git config --global user.email johndoe@example.com
$ git config --global core.editor <EDITOR>
$ git config --list

# Initialize the git tracking
$ git init

# Create .gitignore file with Terminal
$ echo "### ignoring the following files and folders ###" > .gitignore | gedit .gitignore

# Visualize project status (Changes + staged)
$ git status

# Move all changes to staging
$ git add . $ git add <fileName>

# Move specific file to staging
$ git add NUnitTestTriangle.csproj

# Confirm changes to branch With oneline message
$ git commit -m "Chapter 2.04, Domain, Repository, Services"

# confirm changes to branch and open editor for message
$ git commit

# Override last commit message
$ git commit --amend -m "Initial commit, app v0. SNAPSHOT"

# Override last commit and open editor for message
$ git commit --amend

# List all commits in descendent order
$ git log

# Set Gedit as default git message editor
$ git config --global core.editor "gedit -s"

# return a previous version
$ git reset --hard c067ddd817133e50e912b5cd31964dbc315337af

Branching

#Rename branch from master to main
$ git branch -M main

# Differences between branches
$ git diff --stat --color main..<BRANCH_NAME>

# Create new branch
$ git branch new_feature

# Change branch (Move HEAD)
$ git checkout new_feature

# Create new branch and Move HEAD
$ git checkout -b netto_version

# Delete branch
$ git branch -d new_feature

# list remote branches
$ git branch -r
