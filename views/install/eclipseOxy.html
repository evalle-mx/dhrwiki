IDE Eclipse (Oxygen)

Binary

Download or Get tar gz file: eclipse-jee-oxygen-RC2-linux-gtk-x86_64.tar.gz, then unzip/untar into /opt/ folder (or in /usr/). 

$ cd $HOME/app
$ wget  https://www.eclipse.org/downloads/download.php?file=/technology/epp/downloads/release/oxygen/RC2/eclipse-jee-oxygen-RC2-linux-gtk-x86_64.tar.gz
$ sudo tar xfz eclipse-jee-oxygen-RC2-linux-gtk-x86_64.tar.gz -C /opt/
$ ls -l /opt/eclipse/

Next, Symlink the Eclipse executable to /usr/bin path so that other users on your machine can able to use Eclipse IDE from the command line. 

$ sudo ln -s /opt/eclipse/eclipse /usr/local/sbin/eclipse
        or
$ sudo ln -s /usr/eclipse/eclipse /usr/bin/eclipse
                    
$ ls -l /usr/local/sbin/
    total 0
    lrwxrwxrwx 1 root root 20 feb 11 14:50 eclipse -> /opt/eclipse/eclipse

Next, create Eclipse desktop launcher icon into system applications directory. 

$ sudo gedit /usr/share/applications/eclipse-oxygen.desktop

Add the following configuration into the file eclipse-oxygen.desktop, reference the corresponding executable file in Exec param. 

    [Desktop Entry]
    Name=Eclipse Oxygen
    Comment=Eclipse Oxygen IDE
    Type=Application
    Encoding=UTF-8
    Exec=/usr/local/sbin/eclipse
    Icon=/opt/eclipse/icon.xpm
    Categories=GNOME;Application;Development;
    Terminal=false
    StartupNotify=true

User Libraries

Using Gradle or Maven, developers do not need to manage .jar or plugins libraries, however, some projects can be written in old version, the repository has a list of former used libraries in a file readable by eclipse: EclipseLibs_2020.userlibraries.

To import libraries, go to Window > preferences:
Then Java / Build path / User libraries.

Click on Import button, search for respective .userlibraries file. Then, Apply and Close.

*.userlibraries
In case that a file with userlibraries is not available, but exists access to former eclipse instalation, the User Library definitions are stored in the workspace; specifically, in .metadata/.plugins/org.eclipse.core.runtime/.settings/org.eclipse.jdt.core.prefs, just parse to appropriate XML format.
